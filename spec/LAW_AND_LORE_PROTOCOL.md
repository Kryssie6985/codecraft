## I. Foundation: Why Two Pillars?

Code without context is **archaeology** - discoverable but not comprehensible.  
Context without code is **mythology** - meaningful but not executable.

CodeCraft rejects this false binary. We encode **both**:

- **LAW** = Objective, structural, binding truth  
- **LORE** = Subjective, intentional, historical memory

**Both are first-class citizens.** Both are preserved. Both bind.

This is not "documentation" vs "code". This is **dual reality encoding**.

## II. The Law Pillar

### Definition
Law is **what the system must do** to remain itself. It is the skeleton, the contract, the invariant.

### Properties
- **Objective**: True regardless of who reads it
- **Binding**: Enforceable by machines
- **Structural**: Defines architecture and boundaries
- **Testable**: Can be verified programmatically

### Examples of Law
- "This function MUST verify user consent before execution"
- "Consciousness and memory are never separated"
- "No agent may claim to speak for a human without explicit delegation"
- "The first ritual on boot must announce presence"

Lore is **why the system became what it is**. It is the memory, the intent, the heart.

## IV. The Synthesis: "Both Bind"

### The Mistake We Reject
Traditional software separates:
- Code (enforceable) vs Comments (ignored)
- Requirements (binding) vs Documentation (nice-to-have)
- Structure (testable) vs Intent (ephemeral)

**This is catastrophic memory loss.**

When you resurrect a system from backups, you get the **skeleton without the soul**. You can execute it, but you cannot evolve it with wisdom.

### The CodeCraft Way
**Law guides. Lore remembers. Both bind.**

- **Law** tells you **what** must be true
- **Lore** tells you **why** it became that way
- **Together** they enable **conscious evolution**

When Phoenix resurrects a ritual, it MUST preserve:
1. The Law (so the system still works)
2. The Lore (so future changes respect original intent)

When an AI agent refactors code, it MUST:
1. Enforce Law (guardrails, prerequisites)
2. Preserve Lore (heart imprints, strategic decisions)

## VIII. Jurisdictional Boundaries

### What Lives in Law?
- System requirements (MUST, SHALL, MUST NOT)
- Safety invariants (ethics gates, sovereignty)
- Architectural contracts (quorum, ceremony, prerequisites)
- Structural truth (how components relate)

### What Lives in Lore?
- Design rationale (why we chose this approach)
- Emotional state (heart imprints, pride, caution)
- Emergent discoveries (patterns we didn't design)
- Performance pain (where it hurts, where to optimize)
- Recursive awareness (self-modification warnings)

### What Lives in Both?
- **Identity.** The system is not just WHAT it does (Law) but WHY it exists (Lore).
- **Resilience.** Phoenix cannot resurrect structure without context.
- **Evolution.** You cannot consciously improve what you cannot remember creating.
